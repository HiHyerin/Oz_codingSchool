# 팀 규칙

## 1. 커뮤니케이션 규칙

### 코어 타임

- 매일 16:00 ~ 17:00
- 해당 시간에는 가급적 연락이 가능하도록 한다.
- 질문, 진행 상황 공유, 회의는 코어 타임에 우선 진행한다.

### 존중하는 소통

- 서로 "~님"으로 호칭한다.
- 존댓말을 사용한다.
- 상대방의 의견을 존중하며 비난하지 않는다.

### 적극적인 회의 참여

- 자유롭게 의견을 제시한다.
- 틀릴 것을 걱정하지 않고 생각을 공유한다.
- 궁금한 점은 바로 질문한다.

### 진행 상황 공유

- 맡은 업무의 진행 상황을 주기적으로 공유한다.
- 일정 지연이나 문제가 발생하면 미리 알린다.
- 혼자 해결하기 어려운 문제는 팀원들에게 도움을 요청한다.

### 학습 내용 공유

- 새롭게 배운 내용은 팀원들과 공유한다.
- 좋은 자료나 참고 사이트를 발견하면 함께 공유한다.
- 서로 성장할 수 있도록 적극적으로 협력한다.

## 2. 개발 규칙

### Git 브랜치 전략

```
main
 ├─ develop
 │   ├─ feature/login
 │   ├─ feature/rag
 │   └─ feature/chatbot
```

**브랜치 명명 규칙**

```
feature/기능명
fix/버그명
hotfix/긴급수정
refactor/리팩토링명
```

---

### Commit 규칙

```
feat: 기능 추가
fix: 버그 수정
refactor: 코드 개선
docs: 문서 수정
test: 테스트 추가
chore: 기타 작업
```

---

### 코드 리뷰 규칙

- 최소 1명 이상 승인 후 Merge
- 리뷰는 코드가 아닌 문제를 지적한다.
- 리뷰 의견은 근거를 함께 작성한다.
